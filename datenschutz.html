Zweck der App

VetOR ist eine Organisations-App für tiermedizinische OP-Teams. Die App dient zur Verwaltung von OP-Vorbereitungslisten, Bestecklisten, PDF-Unterlagen, Telefonlisten, Nutzerzugriffen und internen Verbesserungsvorschlägen.

Welche Daten werden verarbeitet?

  • Vorname und Nachname
  • E-Mail-Adresse
  • Funktion innerhalb der Klinik
  • Nutzerrolle, zum Beispiel Admin oder Nutzer
  • Zugriffsberechtigungen auf Module
  • interne Vorschläge und Notizen
  • Listenpunkte für OP-Vorbereitungen
  • optional hinzugefügte Bilder
  • optional hinzugefügte PDF-Dateien
  • Klinikdaten, zum Beispiel Klinikname und Adresse

Wofür werden die Daten verwendet?

  • Bereitstellung von Nutzerkonten
  • Nutzerverwaltung durch Admins
  • Anzeige von OP-Vorbereitungen und Bestecklisten
  • Verwaltung interner Vorschläge
  • Anzeige berechtigter App-Inhalte

Lokale Speicherung

Im aktuellen Stand speichert VetOR die Daten lokal auf dem Gerät. Es findet keine automatische Übertragung an einen externen Server statt, solange keine Cloud-, Backend- oder Push-Funktion eingebaut wird.

Keine Werbung und kein Tracking

VetOR verwendet im aktuellen Stand:

  • keine Werbung
  • kein Tracking
  • keine externen Analyse-Tools
  • keine Weitergabe von Daten an Werbenetzwerke

Bilder und Dokumente

Nutzer können optional Bilder zu Vorschlägen hinzufügen. Admins können PDF-Dateien im Besteckordner hinterlegen. Diese Inhalte sollten keine personenbezogenen Patientendaten, Besitzerinformationen oder vertraulichen Informationen enthalten, sofern dies nicht intern geregelt und erlaubt ist.

Konto und Löschung

Nutzer können ihr Konto in der App löschen. Dabei wird das lokale Konto auf dem Gerät entfernt. Falls zukünftig eine Server-Synchronisierung eingebaut wird, muss die Löschung auch serverseitig umgesetzt und diese Datenschutzerklärung angepasst werden.

Sicherheit

Einladungscodes für Admins sind in der App zusätzlich durch Face ID, Touch ID oder den Gerätecode geschützt, sofern das Gerät dies unterstützt. Passwörter werden im aktuellen Stand nicht im Klartext neu gespeichert.

Änderungen

Diese Datenschutzerklärung kann angepasst werden, wenn neue Funktionen eingebaut werden, insbesondere Push-Mitteilungen, Cloud-Synchronisierung, E-Mail-Passwort-Zurücksetzung, externe Server, Analyse- oder Crash-Reporting.
